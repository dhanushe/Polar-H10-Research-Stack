iOS App

Requirements

  • Device: iPhone or iPad running iOS 16+, or a Mac with Apple Silicon (macOS 13+)
  • Hardware: Polar H10 chest strap with a charged battery
  • Bluetooth: Device Bluetooth must be enabled; location permission required for BLE scanning
  • Xcode: 15+ required to build from source (Swift 5.9+)

Quick Start

  1. Build and run the URAP Polar H10 V1 Xcode project on your device.
  2. Open the Dashboard tab and tap + to scan for nearby Polar H10 sensors.
  3. Tap a discovered sensor to connect. The sensor card appears with live HR and RR data.
  4. Tap Start Recording and enter or auto-generate a 20-character session ID.
  5. When finished, tap Stop. The session is saved and appears in the Recordings tab.
  6. To use the Python API, find your Base URL under Settings → API for Python.

Recording IDs

Each session is identified by a 20-character alphanumeric ID that you provide at the start of recording. IDs appear in exported filenames and are used to retrieve sessions via the Python API.

  • Tap Auto-generate to have the app create a random ID.
  • Enter a custom ID (e.g. SUBJECT01SESSION01AB) to match your study protocol.
  • IDs must be exactly 20 characters; the app validates this before starting.

Export Options

Access export options from the Recordings tab → tap a session → Export.

  • CSV Zip: Structured zip archive with one CSV per data stream per sensor. Best for offline Python analysis. See CSV Zip Structure.
  • JSON: Complete session as a single JSON file, using the same schema as the REST API response.
  • Share Sheet: Shares the CSV zip via iOS share sheet (AirDrop, Files, Mail, etc.).
Tip: The Python client's load_from_zip() method parses exported CSV zips offline with no app or network connection required.

Python Client

Installation

# Install dependencies
pip install requests pandas matplotlib

# Or install the package from the repo root
pip install -e python-client/

Python 3.9 or newer is required. The client lives in the python-client/ directory of the repository.

Connecting to the App

The app runs a local HTTP server on port 8080 while it's in the foreground. Your computer and iOS device must be on the same Wi-Fi network.

  1. Open the iOS app and keep it in the foreground.
  2. Go to Settings → API for Python.
  3. Copy the Base URL shown (e.g. http://10.45.216.117:8080).
import urap_polar

base_url = "http://10.45.216.117:8080"  # your device's URL from Settings
Note: If the Base URL section in Settings is empty, ensure the app is in the foreground and the device is connected to Wi-Fi (not cellular).

Working with Recordings

# List all recordings on the device
recordings = urap_polar.list_recordings(base_url=base_url)
for r in recordings:

# Fetch a full session by ID
session = urap_polar.get_recording(recordings[0].id, base_url=base_url)

DataFrames

# All sensors at once
dfs = session.to_dataframes()
# { sensor_id: { "heart_rate": df, "rr_intervals": df } }

# Per-sensor
hr_df = sensor.heart_rate_dataframe()
rr_df = sensor.rr_dataframe()

# Columns: timestamp, value, monotonic_timestamp
print(hr_df.head())

All DataFrames have three columns: timestamp (ISO8601 string), value (BPM for HR, milliseconds for RR intervals), and monotonic_timestamp (seconds since device boot for precise inter-sample timing).

Load from Exported Zip

Analyze exported sessions locally without the app running:

import urap_polar

session = urap_polar.load_from_zip("recording_abc123_csv.zip")

# Same interface as get_recording()
print(session.name, session.id, session.duration_seconds)
for s in session.sensors():
    print(s.sensor_name, s.heart_rate_avg, s.sdnn, s.rmssd)

session.plot(save_path="plot.png")

Batch scripts

  • process_zip_stats.py <zip> [--output report.csv]: Prints session and per-sensor statistics, optionally saving to CSV.
  • zip_to_plot.py <zip> [--save plot.png]: Loads a zip and generates the standard HR + RR figure.
  • verify_zip_parsing.py <zip>: Validates zip structure and reports any issues.

REST API

While the iOS app is open and in the foreground, a local HTTP server on port 8080 serves all recordings on your Wi-Fi network. The base URL is shown in Settings → API for Python.

Availability: The server only runs while the app is in the foreground. Background or suspended state will cause connection refused errors.

List Recordings

GET <base_url>/recordings

# Example
curl http://10.45.216.117:8080/recordings

Returns a JSON array of recording summary objects:

[
  {
    "id": "SUBJECT01SESSION01AB",
    "name": "SUBJECT01SESSION01AB",

    "endDate": "2026-05-01T10:30:00Z",
    "durationSeconds": 1800,
    "sensorCount": 2,
    "averageHeartRate": 72.4,
    "averageSDNN": 48.2,

Get Recording

GET <base_url>/recordings/<recording_id>

# Example
curl http://10.45.216.117:8080/recordings/SUBJECT01SESSION01AB

Returns the full session JSON including per-sensor time series. The response schema mirrors the local JSON export; see JSON Format.

Data Formats

CSV Zip Structure

Exporting a session as CSV Zip produces a zip archive with one folder:

recording_<id>_csv/
├── session_info.csv               # session metadata + sensor summary table
├── sensor_1_<sensorId>_hr.csv
├── sensor_1_<sensorId>_rr.csv
├── sensor_1_<sensorId>_acc.csv
├── sensor_1_<sensorId>_statistics.csv
├── sensor_2_<sensorId>_hr.csv
└── ...

Sensors are numbered from 1. Files are always present even if that sensor recorded no data of that type (they will be header-only).

HRV Metrics

SDNN

Standard Deviation of Normal-to-Normal intervals. Reflects overall heart rate variability, influenced by both sympathetic and parasympathetic nervous system activity. Higher SDNN generally indicates greater autonomic flexibility.

Formula: SDNN = √( (1/N) × Σ(RRᵢ − RR̄)² )

RMSSD

Root Mean Square of Successive Differences. Captures beat-to-beat variability and primarily reflects parasympathetic (vagal) nervous system activity. Commonly used in stress and recovery research.

Formula: RMSSD = √( (1/(N-1)) × Σ(RRᵢ₊₁ − RRᵢ)² )

Analysis Windows

Both SDNN and RMSSD are calculated over a configurable trailing window. Set via Settings → HRV Window in the app:

WindowUse case
1 minuteShort-task studies, rapid state changes
2 minutesGeneral monitoring
5 minutesStandard clinical HRV window (recommended)
10 minutesResting baseline, longer protocol segments

Accelerometer DSP Pipeline

Raw 3-axis data from the Polar H10 is processed on-device before storage:

  1. Sampling: 25 Hz raw samples from the H10 (X, Y, Z axes in milli-g).
  2. High-pass filter (per axis): Biquad IIR at 0.25 Hz removes the static gravity component, leaving only dynamic movement.
  3. Low-pass filter (per axis): Biquad IIR at 5 Hz removes high-frequency sensor noise.
  4. Vector magnitude: |a| = √(x² + y² + z²) computed for each sample.
  5. 1-second averaging: 25 samples are averaged into a single magnitude value per second.

The output is compatible with ENMO (Euclidean Norm Minus One) and HPFVM (High-Pass Filtered Vector Magnitude) activity count metrics used in occupant behavior and built environment research.
