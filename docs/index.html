Capabilities

Everything you need
for HR research

From Bluetooth capture to pandas DataFrames. No proprietary software, no cables, no lock-in.

📡

Multi-Sensor Recording

Connect multiple Polar H10 sensors simultaneously over Bluetooth LE and capture all of them in a single timestamped session. Each sensor gets its own HR, RR, and accelerometer stream.

💓

HRV Analysis

On-device SDNN and RMSSD calculations with configurable 1, 2, 5, or 10-minute windows. Updates continuously in real time.

🗂️

Flexible Export

Structured CSV zips with per-sensor files for HR, RR, and accelerometer data, plus full JSON. Share via AirDrop, Files, or email.

🐍

Python HTTP API

While the app is open, a local server on port 8080 exposes all recordings over your Wi-Fi. Fetch sessions as pandas DataFrames in three lines.

import urap_polar session = urap_polar.get_recording(id, base_url=url) session.plot() # HR + RR for all sensors
📊

Live Charts

Interactive time-range charts (2 min to 1 hour), live HRV readouts, and a real-time 3D accelerometer visualizer.

⚡

Accelerometer DSP

On-device IIR filter pipeline with a 0.25 Hz high-pass and 5 Hz low-pass stage, outputting 1-second activity magnitudes compatible with ENMO/HPFVM metrics.

Workflow

Three steps from
chest strap to DataFrame

The full pipeline from Bluetooth capture to publication-ready figures.

1

Connect & Record

Open the Dashboard, pair your Polar H10 over Bluetooth, enter a 20-character session ID, and tap Record. Live HR and RR data stream immediately to the screen.

2

Export or Stream

Stop the recording and export as a CSV zip for offline analysis, or keep the app open to stream sessions live to Python over your local Wi-Fi network using the built-in HTTP API.

3

Analyze in Python

Use the included urap_polar client to fetch sessions, build pandas DataFrames, compute HRV, and generate publication-ready matplotlib figures. All in a few lines of code.

import urap_polar

# Base URL from: iOS app → Settings → API for Python
base_url = "http://10.45.216.117:8080"

# List all sessions on device
recordings = urap_polar.list_recordings(base_url=base_url)
for r in recordings:
    print(r.id, r.name, r.duration_seconds)

# Fetch a full session and plot HR + RR for all sensors
session = urap_polar.get_recording(recordings[0].id, base_url=base_url)
session.plot(save_path="recording.png")
import urap_polar

# Load an exported CSV zip without the app or network
session = urap_polar.load_from_zip("recording_abc123_csv.zip")

print(session.name, session.id, session.duration_seconds)
for s in session.sensors():
    print(s.sensor_name, s.heart_rate_avg, s.sdnn, s.rmssd)

session.plot(save_path="output.png")
# Export all sensor data to pandas DataFrames
dfs = session.to_dataframes()
# → { sensor_id: { "heart_rate": df, "rr_intervals": df, "accelerometer": df } }

sensor = session.sensors()[0]
hr_df  = sensor.heart_rate_dataframe()   # columns: timestamp, value, monotonicTimestamp
rr_df  = sensor.rr_dataframe()

# HRV stats (per-sensor)
print(sensor.sdnn, sensor.rmssd, sensor.hrv_window)

About

Built for rigorous
physiological research

This project is part of an Undergraduate Research Apprenticeship Program (URAP) at the Center for the Built Environment, UC Berkeley, investigating how indoor environmental conditions affect occupant physiology.

The Polar H10 chest strap provides clinical-grade ECG-derived HR and RR intervals. This stack captures that signal, adds accelerometer context for activity estimation, and delivers it to Python with the same interface whether you're streaming live or analyzing an exported zip offline.

The dual architecture (Swift app for capture, Python client for analysis) means researchers can focus on their study design, not data plumbing.
